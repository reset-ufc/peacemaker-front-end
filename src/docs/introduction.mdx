# Ajuda & Onboarding do Peacemaker Bot

Bem-vindo! Aqui você vai aprender, passo a passo, como começar a usar o **Peacemaker Bot** para monitorar e manter a civilidade nos seus repositórios GitHub. Sinta-se à vontade para navegar nesta página toda vez que tiver dúvidas.

---

## 1. Instalar o Bot no GitHub

O primeiro passo é adicionar o Peacemaker Bot como um _GitHub App_ no seu repositório:

- Clique no link de instalação: [Instalar Peacemaker Bot](https://github.com/apps/peacemakerapppoc)
- Autentique-se no GitHub e escolha o(s) repositório(s) onde o bot terá permissão para atuar.

> **Dica:** você pode instalar em mais de um repositório, mas lembre-se de escolher apenas aqueles que deseja monitorar.

---

## 2. Primeiro comentário incivilizado

Assim que alguém (inclusive você!) fizer um comentário com linguagem considerada _incivilizada_ num dos repositórios permitidos, o bot irá:

1. Detectar o comentário e calcular um **score de toxicidade**.
2. Gerar sugestões de reformulação para tornar o texto mais amigável.
3. Exibir, na aba **Incivilities**, o comentário original, a classificação de incivilidade e as sugestões.

Na aba **Incivilities**, você tem três opções para cada sugestão:

- **Aceitar**: confirma a sugestão e substitui o texto no GitHub.
- **Editar**: ajusta a sugestão manualmente antes de aplicar.
- **Recusar**: descarta aquela sugestão e gera uma nova.

---

## 3. Configurando seu Token Pessoal do GitHub

Antes de aceitar qualquer sugestão automática, você precisa fornecer um **Personal Access Token** (PAT) do GitHub com permissão para modificar comentários. Sem ele, o bot não conseguirá substituir o texto.

1. Acesse sua conta no GitHub e vá em: **Settings > Developer settings > Personal access tokens**.
2. Clique em **Generate new token** e selecione o escopo **`repo`** (ou apenas **`public_repo`**, se for um repositório público).
3. Copie o token gerado e salve-o em um local seguro.
4. No painel do Peacemaker Bot, abra **Configurações** (ícone de engrenagem no perfil) e cole seu token.

> **Link útil:** [Documentação oficial de PATs do GitHub](https://docs.github.com/pt/authentication/keeping-your-account-and-data-secure/creating-a-personal-access-token)

---

## 4. Edição de sugestões & reavaliação

Quando você **edita** e submete uma sugestão:

1. O bot reanalisa o texto editado em busca de traços de incivilidade.
2. Existem dois caminhos possíveis:

   **A. Ainda incivilizado**

   - Você vê um alerta informando que o texto ainda não está ok.
   - Novas sugestões são geradas para melhorar novamente.
   - Se, após **duas edições**, ainda houver incivilidade, o bot para de sugerir e exibe um alerta final: agora é você quem precisa reformular sem ajuda automática.

   **B. Resolvido**

   - O bot não detecta mais incivilidade e marca o comentário como _resolvido_.
   - Parabéns! A conversa está num tom saudável.

> **Atenção:** esse fluxo evita que você fique num ciclo infinito de edições sem sair do lugar.

---

## 4. Explorando a Dashboard

Por fim, visite a **Dashboard** para acompanhar métricas e relatórios:

- **Filtros de período**: escolha entre últimos 24h, 7d, 30d ou 1 ano.
- **Visão geral**: cartões com média/mediana de toxicidade, total de comentários, itens resolvidos, etc.
- **Gráficos interativos**: atividade de moderação, incivilidades por tipo (issues vs PRs), categorias de incivilidade e ações tomadas.
- **Filtro por repositório**: foque em um projeto específico ou visualize todos de uma vez.

Use esses dados para entender tendências, treinar sua equipe e manter a comunidade engajada num ambiente respeitoso.
